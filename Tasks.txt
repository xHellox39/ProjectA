After comparing the frontend and backend archives, there are some significant integration problems in addition to the backend issues already identified.

# 🔴 Critical Frontend ↔ Backend Integration Issues

## 1. API Base URL Mismatch (Major)

### Frontend File

```js
src/api/ApiClient.js
```

Uses:

```js
http://localhost:3500/api
```

### Another Frontend File

```js
src/services/api.js
```

Uses:

```js
http://127.0.0.1:8000/api/v1
```

### Backend

Routes are mounted as:

```txt
/auth
/users
/properties
/bookings
/payments
/maintenance
/communication
/admin
/reports
```

There is **no**:

```txt
/api
/api/v1
```

prefix in the backend.

### Result

Many frontend requests will likely hit:

```txt
404 Not Found
```

### Fix

Choose one:

```txt
http://localhost:3500
```

or

```txt
http://localhost:8000
```

and use it everywhere.

---

# 2. Two Different API Systems

Frontend contains:

```txt
src/api/ApiClient.js
src/services/api.js
```

Both define API communication.

### Risk

Different pages may call:

```txt
localhost:3500
```

while others call:

```txt
localhost:8000
```

leading to inconsistent behaviour.

### Fix

Keep only one API client.
